## システム概要

物件掲載写真の加工・保存作業を自動化するツールです。  
Photoshopの手動作業を置き換え、以下を自動で行います。

- カメラRAW補正（露出・コントラスト・ハイライト等）
- ロゴの合成（通常写真のみ／45%不透明度）
- THETA360度画像の自動判別（ロゴなし）
- SUUMOフォルダ構造の自動作成
- ファイル名の自動付与（管理番号_物件名_部屋_画像種別_連番）
- 処理ログの保存
- 失敗ファイルのみの再処理

### 手順1 — Pythonのインストール

1. [https://www.python.org/downloads/](https://www.python.org/downloads/) からPython 3.11をダウンロード
2. インストーラーを起動

コマンドプロンプトを開き、アプリのフォルダに移動してから以下を実行：

```
pip install -r requirements.txt
```

### 手順3 — 動作確認

```
python test_setup.py
```

「✅ All systems ready!」と表示されれば準備完了です。

## Google Sheets の設定（初回のみ・IT担当者が実施）

### 手順1 — サービスアカウントの作成

1. [Google Cloud Console](https://console.cloud.google.com/) を開く
2. **「IAM と管理」→「サービスアカウント」** をクリック
3. **「サービスアカウントを作成」** → 名前を入力（例：`rent-magazine-processor`）→「作成して続行」
4. ロール付与はスキップ → 「完了」
5. 作成したサービスアカウントをクリック → **「キー」タブ → 「鍵を追加」→「新しい鍵を作成」→「JSON」**
6. `.json` ファイルがダウンロードされる → アプリのフォルダに保存する

### 手順2 — APIの有効化

Google Cloud Console → **「APIとサービス」→「ライブラリ」** で以下を有効化：
- **Google Sheets API**
- **Google Drive API**

### 手順3 — 物件管理番号マスターシートの作成

Googleスプレッドシートで新規シートを作成し、**1行目に以下のヘッダーを入力**してください（大文字・小文字・スペースを正確に合わせること）：

```
Management Number | Type | Property Name | Building | Room/Unit | City/Ward | Hiragana Group | Nearest Station | WordPress Post ID | WordPress URL | Status | Notes
```

**管理番号の形式：**

| 種別 | 形式 | 例 |
|------|------|----|
| 居住用 | RM-R000001 | RM-R000001 |
| テナント | RM-T000001 | RM-T000001 |
| ビル・建物 | RM-B000001 | RM-B000001 |

**Hiragana Group（名古屋のみ）：** 物件名の読み仮名グループ（例：「さ」「な」「や」）  
**Nearest Station（名古屋のみ）：** 最寄り駅名（フォルダ名に使用）  
名古屋以外の物件はこの2列を空白にしてください。

### 手順4 — シートの共有

1. スプレッドシートの「共有」ボタンをクリック
2. サービスアカウントのメールアドレス（`...@...iam.gserviceaccount.com`）を貼り付け
3. 権限を「**閲覧者**」に設定 → 「送信」

### 手順5 — アプリへの接続

アプリを起動し、以下の手順で接続してください（「アプリの使い方 手順1」を参照）。

## 基本的な使い方

### 手順1 — Google Sheets に接続する

1. 「認証情報ファイル」の**「参照」**ボタンをクリック
2. IT担当者から受け取った `.json` ファイルを選択
3. 「シート名」に **「物件管理番号マスター」** と入力されていることを確認
4. **「接続」** ボタンをクリック

接続が成功すると「● 接続済み」と表示され、物件ドロップダウンが使えるようになります。

> **接続しない場合は？**  
> Sheetsに接続しなくても手入力モードで使えます。物件名・部屋番号・管理番号・市区などを直接入力してください。

### 手順2 — 物件と部屋を選ぶ

1. **「物件名」** ドロップダウンから物件を選択
2. **「部屋番号」** ドロップダウンから部屋を選択
3. 管理番号・市区・最寄駅が自動で表示されます

### 手順3 — 画像種別を入力する

**「画像種別」** フィールドに、今回処理する写真の場所を入力します。

| 入力例 | 説明 |
|--------|------|
| リビング | リビングルームの写真 |
| 寝室 | 寝室の写真 |
| 玄関 | 玄関の写真 |
| キッチン | キッチンの写真 |
| 洗面所 | バスルーム・洗面所の写真 |
| 外観 | 建物外観の写真 |

> THETA360度写真は自動判別され「THETA」と命名されます。画像種別の入力は不要です。

### 手順4 — ファイルを選ぶ

| 項目 | 説明 |
|------|------|
| **入力フォルダ** | 処理する写真が入ったフォルダ |
| **ロゴファイル** | Rent Magazineのロゴ画像（PNG）— 一度選ぶと次回から記憶 |
| **出力フォルダ** | 処理済み写真の保存先 — 一度選ぶと次回から記憶 |

### 手順5 — 処理を開始する

**「▶ 処理開始」** ボタンをクリックします。

- プログレスバーがリアルタイムで進行状況を表示します
- 「✅ 成功」パネルに成功したファイルが表示されます
- 「❌ エラー」パネルに失敗したファイルとエラー内容が表示されます
- 完了すると「完了」ダイアログが表示されます

### 手順6 — 失敗したファイルを再処理する

処理後に失敗ファイルがあると **「↺ 失敗ファイルを再処理（N件）」** ボタンが有効になります。  
クリックすると失敗したファイルだけを再処理します（全ファイルの再実行は不要）。

## ファイル命名規則

```
管理番号_物件名_部屋番号_画像種別_連番.jpg
```

**例：**
```
RM-R000001_八ツ田ハイツ北棟_101_リビング_01.jpg
RM-R000001_八ツ田ハイツ北棟_101_リビング_02.jpg
RM-R000001_八ツ田ハイツ北棟_101_THETA_01.jpg
```

- 連番は出力フォルダの既存ファイルを数えて続きから付与（再実行しても重複しません）
- 管理番号が未入力の場合はその部分が省略されます

## トラブルシューティング

### アプリが起動しない

```
python test_setup.py
```
を実行して不足ライブラリを確認してください。

---

### 「PyQt5が見つかりません」

```
pip install PyQt5
```

---

### 「SpreadsheetNotFound」エラー

- シート名が正確か確認（「物件管理番号マスター」）
- シートがサービスアカウントのメールアドレスと共有されているか確認

---

### 「Missing required columns」エラー

シートの1行目のヘッダーが正確でない可能性があります。  
「手順3 — 物件管理番号マスターシートの作成」を再確認してください。

---

### ⚠ よみがな・最寄駅がシートに未登録

名古屋の物件で「Hiragana Group」「Nearest Station」列が空です。  
シートに入力して「更新」ボタンを押してください。

---

### THETA画像にロゴが合成されてしまう

THETA画像の自動判別は次の順で行われます：
1. EXIFメタデータ（RICOH/THETAメーカータグ）の確認
2. アスペクト比 2:1（横：縦）の確認

いずれにも該当しない場合は通常写真として処理されます。

---

### 処理が遅い

1枚あたり約3〜5秒かかります。10枚で約40〜50秒が目安です。  
処理中は他の重い作業を避けてください。

---

## 自動化されないこと（手動作業）

以下は引き続きスタッフが手動で行います：

- 掲載する写真の選定・並び順の決定
- サムネイル（アイキャッチ）の選択
- THETA写真の掲載位置の確認
- 最終確認・SUUMOへの公開
